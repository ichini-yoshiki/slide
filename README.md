# ichini スライド（HTML プレゼン）

このリポジトリは、**ブラウザで表示するスライド資料**です。本文の「正本」は各デック内の `slides.md` にあり、リポジトリ直下の Python スクリプトで `slide.html` に反映します。

## 何ができるか

- **見る**：デックフォルダ内の `slide.html` をブラウザで開くだけで、スライドをページ送りできます。
- **編集する**：そのデックの `slides.md` を書き換え、ビルドを実行すると、表示用の HTML が更新されます。

初心者の方は、まず **「スライドを見る」** だけでも問題ありません。

---

## フォルダの場所

プレゼンごとに **`SLIDE/<デック名>/`** というフォルダを用意します。例として次の資料が入っています。

```
SLIDE/2026_04_16_PPP/
```

新しい回の資料は、`SLIDE/` の下にフォルダを追加して整理します（詳しくは [SLIDE/README.md](SLIDE/README.md)）。

### GitHub Pages で開くとき

このリポジトリを GitHub Pages（`https://ichini-yoshiki.github.io/slide/`）で見る場合も、パスは **`SLIDE/` を含む**形です。

例（2026_04_16_PPP）:

- 正しい URL: `https://ichini-yoshiki.github.io/slide/SLIDE/2026_04_16_PPP/slide.html`
- 旧 URL（`/2026_04_16_PPP/slide.html` のみ）は、フォルダ移動のため 404 になります。ブックマーク用に [2026_04_16_PPP/slide.html](2026_04_16_PPP/slide.html) から新しい場所へ転送するページを置いています。

## スライドを見る（インストール不要）

1. エクスプローラーで `SLIDE/2026_04_16_PPP` フォルダを開きます。
2. **`slide.html`** をダブルクリックする（または、ブラウザにドラッグ＆ドロップする）。

通常のプレゼンと同じように操作できます。

| 操作 | 方法 |
|------|------|
| 次のスライド | 画面下の **「次へ →」** ボタン、またはキーボードの **→** |
| 前のスライド | **「← 前へ」**、または **←** |
| 一覧でジャンプ | **「一覧」** を押すとサムネイルが開き、クリックで移動 |
| 全画面 | **「全画面」**（もう一度押すと解除） |
| 目次から移動 | 目次スライドの項目をクリック |
| 画像を大きく | スライド内の画像をクリック（**Esc** で戻る） |

一覧が開いているときは **Esc** で閉じられます。

## 内容を編集して HTML を更新する

スライドの文言や枚数を変えるときは、**対象デック**の `slides.md` を編集したあと、次の **ビルド** を実行します。

### 1. 用意するもの

- **Python 3** がパソコンに入っていること  
  - コマンドプロンプトまたは PowerShell で `python --version` と打ち、バージョンが表示されれば大丈夫です。

### 2. 必要なライブラリを入れる（初回だけ）

PowerShell または コマンドプロンプトで、**このリポジトリのルート**に移動してから次を実行します。

```powershell
cd <このリポジトリのルート>
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

（コマンドプロンプトで仮想環境を有効にする場合は `.\.venv\Scripts\activate.bat` です。）

※ venv を使わず、グローバルに `pip install PyYAML` だけ入れても動きますが、他のプロジェクトと競合しにくいので **仮想環境（上の `.venv`）を推奨**します。

### 3. ビルドを実行する

ビルド対象の **デックのフォルダ**を引数で渡します。

```powershell
cd <このリポジトリのルート>
.\.venv\Scripts\Activate.ps1
python build_slides.py SLIDE/2026_04_16_PPP
```

成功すると、例のようなメッセージが出て、そのデック内の **`slide.html` が上書き更新**されます。

```
patched: ...\slide.html slides: 11 title: ...
```

その後、ブラウザで該当フォルダの `slide.html` を開き直すか、**再読み込み（F5）** すると変更が反映されます。

## `slides.md` の書き方（かんたんガイド）

- ファイルは **YAML の「複数ドキュメント」** です。スライド **1 枚ごと** に `---` で区切ります。
- 各ブロックに `type:` を書き、種類に応じて項目を埋めます。

| `type` | 意味 |
|--------|------|
| `cover` | 表紙 |
| `toc` | 目次（項目数は、あとに続く `section` の数と同じにする） |
| `section` | 章の区切り（SECTION 見出し） |
| `bullets` | 箇条書きスライド（直前に必ず `section` が必要） |
| `cards` | カード型まとめ（直前に必ず `section` が必要） |

- **画像を使う**（`type: bullets` のとき）：`image: ファイル名.png` のように、`slide.html` と同じフォルダ内のパスで指定します。
- **リンクを置きたい**ときは、箇条書きの1行を `- html: '<a href="...">...</a>'` のように書けます（`build_slides.py` の仕様）。

詳しいルールや注意点は、[build_slides.py](build_slides.py) 冒頭のコメントにもまとまっています。

## よくあるつまずき

| 現象 | 対処 |
|------|------|
| `PyYAML が必要です` と言われる | リポジトリ直下で `pip install -r requirements.txt` を実行したか確認する |
| ビルドで「目次の項目数と section の枚数が一致しません」 | `toc` の `items` の行数を、`section` の枚数と揃える |
| 画像が表示されない | `image:` のファイル名がフォルダ内の実在ファイルと一致しているか、`slide.html` と同じ場所にあるか確認する |

## ファイルの役割（一覧）

### リポジトリ直下

| ファイル | 役割 |
|----------|------|
| [build_slides.py](build_slides.py) | 各デックの `slides.md` から `slide.html` のスライド部分を生成する |

|----------|------|
| `slides.md` | **編集する原稿**（スライドの中身の正本） |

| `img01.png` など | スライド用の画像（`slides.md` の `image:` から参照） |
